# 0.1.0 — 2026-05-30

## Added

- **`osv_query`** — single-package vulnerability lookup against the Google OSV.dev API; returns CVSS vectors, CVE aliases, version ranges, first-safe versions, and severity labels.
- **`osv_query_batch`** — batch dependency audit up to 1000 packages per call with parallel per-package fan-out; each finding exposes CVE `aliases` for direct chaining to `nist-nvd-mcp-server` (`nvd_get_cve`) for CVSS base scores, EPSS exploitation probability, and CISA KEV status; results ≥ 200 packages spill to DataCanvas (`osv_batch_results` table) for SQL aggregation.
- **`osv_get_vulnerability`** — full advisory record fetch by OSV ID (GHSA, PYSEC, RUSTSEC, GO, DSA, CVE); returns summary, full details Markdown, all affected packages and version ranges, CVSS vectors, CWE IDs, and upstream references.
- **`osv_list_ecosystems`** — static list of 26 supported ecosystem identifier strings (case-sensitive exact matches required by the OSV API); drives input validation in `osv_query` and pre-flight rejection in `osv_query_batch`.
- Keyless access to OSV.dev — no API key required; only optional `OSV_REQUEST_TIMEOUT_MS` (default 10000 ms) configures behavior.
- Typed error contracts on all tools — `invalid_ecosystem`, `vulnerability_not_found`, `batch_too_large` with typed `ctx.fail()` and agent-actionable recovery hints.
- STDIO and Streamable HTTP transports via `@cyanheads/mcp-ts-core` ^0.9.16.
